一、数制的概念

1. 数制的定义

数制，又称"计数制"，指用一组固定的符号和统一的规则来表示数值的方法。

数制关注的内容：每一位的构成（使用什么符号即数码）和进位的规则。

2. 基本概念

  • 基数 —— 使用的数码的个数，即每位最多能用多少个符号来表示。基数是 N 则称为 N 进制，即"逢 N 进一"。
  • 权 —— 每一位的 1 代表的大小，即 Ni（N 为基数，i 为位序号）。

3. 常用数制

数制基数数码逢几进一后缀标记
二进制20, 1逢2进1B
八进制80~7逢8进1O / Q
十进制100~9逢10进1D
十六进制160~9, A~F逢16进1H
二、数制间的转换

1. 任意进制 → 十进制（按权展开法）

只需要按照 D = Σ ki × Ni 展开即可。其中 ki 为第 i 位的系数，Ni 为第 i 位的权，N 为基数。

例：(1011.01)2 = 1×2³ + 0×2² + 1×2¹ + 1×2⁰ + 0×2⁻¹ + 1×2⁻² = 8+2+1+0.25 = 11.25

2. 十进制 → 二进制

整数部分：反复除 2 取余数，从右向左排列（从低位到高位排列）。

小数部分：反复乘 2 取整数部分，从左向右排列（从高位到低位排列）。

例：(173.8125)10 → 二进制
整数部分：173 ÷ 2 = 86…1, 86 ÷ 2 = 43…0, 43 ÷ 2 = 21…1, 21 ÷ 2 = 10…1, 10 ÷ 2 = 5…0, 5 ÷ 2 = 2…1, 2 ÷ 2 = 1…0, 1 ÷ 2 = 0…1 → 10101101
小数部分：0.8125×2=1.625 取1, 0.625×2=1.25 取1, 0.25×2=0.5 取0, 0.5×2=1.0 取1 → .1101
结果：(10101101.1101)2

3. 二进制 ↔ 八进制 / 十六进制

二→八：从小数点向两边，每 3 位一组，不足补 0，每组转成一个八进制数码。

二→十六：从小数点向两边，每 4 位一组，不足补 0，每组转成一个十六进制数码。

反过来：八/十六→二，每一位展开成对应的 3/4 位二进制即可。

例：(01011110 10110010)2
→ 每4位一组：0101 1110 1011 0010
→ (5EB2)16
三、二进制算术运算

1. 运算特点

  • 和十进制数运算的规则基本相同，区别在于"逢二进一"。
  • 乘法和除法可以转化为移位操作与加减操作。
  • 通过补码运算（即将引入的概念），可以将减法运算转换为加法运算。因此，数字电路普遍采用二进制进行算术运算。

2. 原码、反码、补码

表示正负：在前面增加一个符号位，0 表示正数，1 表示负数。符号位的 0/1 不代表大小，只是代表正负。

转换关系：

  • 正数（符号位为 0）：原码 = 反码 = 补码
  • 负数（符号位为 1）：原码 → 反码为所有数字位取反，补码等于反码加 1
  • 补码的补码即为原码
易错点：
① 反码取反是指对数字位取反，不改变符号位！例如 -7 即 1111 的反码是 1000，不是 0000。
② 补码是不对称的：例如补码为 1000 代表 -8，而不代表 -0！即同样位数的二进制补码，负数会比正数"多一个"。
③ 补码的"加1"指的是在最低位加 1 的二进制加法运算，不是指数值 1。如果含有小数部分，1 是加在整体的最右边。

3. 补码加法运算步骤

利用二进制补码运算求解带符号数的加法运算：

  1. Step 1：分析需要使用多少个数字位（不能忽略！）
  2. Step 2：用带符号位的二进制数表示，求解每个二进制数的补码
  3. Step 3：对补码进行加法运算，再对结果求补码，即为最终结果
易错：注意最后一步，对补码进行加法运算时，数字位最高位产生的进位要进到符号位上去！（教材中描述的"舍弃产生的进位"指的是舍弃符号位产生的进位）
例：用补码计算 13 - 10
分析：13 和 10 都 < 16，需要 4 个数字位加 1 个符号位
+13 补码：0 01101，-10 补码：1 10110
相加：0 01101 + 1 10110 = 0 00011（符号位进位舍弃）
结果补码 0 00011 → 正数，即 +3 ✓

4. 溢出判断

  • 两个无符号数相加，如果最高位产生进位输出，则肯定发生溢出。
  • 两个最高位不同的补码进行相加运算，肯定不会产生溢出。
  • 两个补码进行相加运算，如果最高位产生进位输出，不一定发生溢出（区别于无符号数！）。
  • 两个补码的减法运算可以用加法器来实现。
例3 (1)：已知 A 的原码为 011010，则 2A 对应的 8 位原码形式为 00110100，-A 的 8 位补码为 11100110。
例3 (2)：10 位二进制补码能够表示的数值范围是 +511 ~ -512。（2⁹ = 512）

2. 格雷码（循环码）

核心特点：相邻的代码只有一位改变状态。这是格雷码最大的特点和优点。

应用：减少代码转换过程中的过渡噪声。

3. 二进制数与格雷码的转换

二进制 → 格雷码：保留最高位不变，从左边开始，依次将每一位二进制码中的值与右边一位二进制码中的值异或（⊕），作为右边一位格雷码中的值。

例：自然二进制码 1 0 1 1 0
                        ↓ ⊕ ⊕ ⊕ ⊕
格雷码：          1 1 1 0 1

格雷码 → 二进制：保留格雷码的最高位作为自然二进制码的最高位，从左边开始，依次将每一位二进制码中的值与右边一位格雷码中的值异或，即二进制转化至格雷码的逆运算。

例：格雷码     1 1 1 0 1
                        ↓ ⊕ ⊕ ⊕ ⊕
自然二进制码：1 0 1 1 0

2. 复合逻辑运算

与非 —— "先与再非"，Y = (A·B)'

或非 —— "先或再非"，Y = (A+B)'

与或非 —— "先与再或再非"，Y = (A·B + C·D)'

异或 ⊕ —— 不同时输出 1，相同时输出 0

同或 ⊙ —— 相同时输出 1，不同时输出 0

异或和同或两种运算关系是对立（互补）的，即异或取反即为同或。

异或和同或都可以用基本的与、或、非运算关系表示：

Y = A ⊕ B = A'B + AB'
Y = A ⊙ B = AB + A'B'

3. 与、或、非、异或的位操作用途

  • 与运算 AND：可以用来将指定的位置 0。任何变量和 0 相与都为 0，任何变量和 1 相与不改变此变量。
    例：想将 10111101 的第五位和第六位（从右往左）置零，则可以 (10111101) AND (11001111)。
  • 或运算 OR：可以用来将指定的位置 1。和与运算刚好相反，任何变量和 0 相或都不改变此变量，任何变量和 1 相或都为 1。
  • 非运算 NOT：用于取反，但是只能同时将整体取反，不能将指定位取反。
  • 异或运算 XOR：用于指定位的取反。任何变量与 0 异或都不改变此变量，任何变量与 1 异或得到的是反变量。
    例：想将 10111101 的低四位取反，则可以 (10111101) XOR (00001111)。

七、逻辑函数及其表示方法

1. 五种表示方法

  • 真值表 —— 遍历输入变量所有取值下的输出值，列成表格
  • 逻辑函数式 —— 将输出写成输入变量的与、或、非等逻辑运算的组合式
  • 逻辑图 —— 逻辑函数式用图形符号表示
  • 波形图 —— 输入变量所有取值可能与对应输出的时域波形
  • 卡诺图（见下一节）

2. 从真值表得到逻辑函数式（重点）

  1. Step 1：找出真值表中使逻辑函数输出 Y = 1 的输入变量取值的组合即使 Y = 1 的行
  2. Step 2：使 Y = 1 的每组输入变量取值的组合（即每一行）对应一个乘积项，取值为 1 写入原变量，取值为 0 写入反变量
  3. Step 3：将各行对应的乘积项相加，即最终结果为"与或"形式
从真值表得到逻辑函数式是几种不同表示方法转换中最重要的一个，因为在后面数字电路的设计学习中，很多时候输入和输出的逻辑关系比较复杂，不能直接看出逻辑函数式，这个时候可以通过遍历得到真值表，再将真值表转化为对应的逻辑函数式。

3. 最小项

特点：

  1. 全体最小项之和为 1
  2. 各个最小项之间是互斥的，即任何两个最小项之积为 0；换句话说，对于任何一个特定的输入变量取值，有且仅有一个最小项为 1
  3. 两个相邻的最小项之和可以合并，消去一对因子，只留下公共因子；这里的"相邻"不是按照自然二进制数编号后的相邻，而是指逻辑上的相邻，规定仅一个变量不同的最小项彼此称为相邻

4. 最大项

定义：M 是相加项，包含 n 个因子，n 个变量均以原变量和反变量的形式在 M 中出现一次。

可以把最大项看作是对每一个最小项应用反演定理，因此，最大项的编号原则是反的：即原变量记为 0，反变量记为 1，n 位自然二进制数排列后对应的十进制数为编号。

例如 A+B+C' 的编号是 001 即 M₁
最小项 mᵢ 与 最大项 Mᵢ 互补：mᵢ = Mᵢ'

5. 最小项之和与最大项之积的转换

例：Y = A'B + AC

展开为最小项之和：
Y = A'B(C+C') + A(B+B')C = A'BC + A'BC' + ABC + AB'C
= m₃ + m₂ + m₇ + m₅ = Σm(2,3,5,7)

转换为最大项之积：
直接变换得到最大项之积过程比较繁琐，因通常应用最小项解决问题！
缺失的最小项编号 → 对应的最大项编号：ΠM(0,1,4,6)
八、逻辑函数的化简

1. 化简的目标

化简为最简与或式：使用最少的与门、或门和非门来实现逻辑函数。

例：Y = AB'C + ABC' + ABC（两个非门，三个与门，两个或门）
化简 → Y = AB + AC（两个与门，一个或门）
或者 → Y = A(B + C)（一个与门，一个或门）

2. 公式化简法

使用前面学习的基本公式和常用公式；对基础要求较高，没有固定的步骤，难以确定结果正确与否，不通用但有时很快速好用。

对公式化简法，要求掌握但不必深入研究，熟悉教材上的例题和课后题即可。

一个可能会用到公式化简法的情景是：当逻辑函数含有变量数很多时，即很难直接采用卡诺图去进行化简时，一般题目的设计会使解题时经常需要用 AB + A'C + BCD = AB + A'C 这个公式来消去多余的变量，接着再用卡诺图去进行化简。
